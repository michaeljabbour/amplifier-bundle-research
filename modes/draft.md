# Mode: /draft

**Purpose:** Write the actual document in the right structure and tone for the target audience.

**Entrypoint:** after `/critique` in standard recipes (Persona A/B).

**Default agents:** `technical-writer`, `figure-designer`, `citation-manager`

---

## Contract

**Input:** pre-registration + results + critique artifact. Optional: existing draft sections (e.g., Persona A may have written an intro already).

**Output:** complete draft in the appropriate structure (IMRAD for empirical papers, patent-brief skeleton for Persona A, policy-brief skeleton for policy work). Publication-quality figures, tables, and citations integrated inline.

**Exit condition:** complete draft ready for formatting in `/publish`.

## Persona-aware behavior

**Persona A (non-scientist — patent/policy):**
- **Output structure varies by recipe:**
  - Patent brief: Field of Invention → Background → Summary → Detailed Description → Claims → Abstract → Drawings
  - Policy brief: Executive Summary → Problem Statement → Evidence → Recommendations → Appendix
  - White paper: Executive Summary → Introduction → Methodology → Findings → Discussion → Recommendations

- **Language:** professional but plain. No citations yet (those come in `/publish`), but all evidence points are marked with `[cite: reference-key]` for the citation-manager to resolve.
- **Figures:** 2–4 per brief, generated via `figure-designer`. Diagrams for patent context, charts for policy evidence.
- **Tone:** authoritative but accessible. Assume the reader is a busy executive or patent examiner, not a domain expert.
- **Integration:** pulls the pre-registration (as "Methodology" or "Search Strategy" section), pulls critique resolutions (noted in margins as revisions), pulls evidence summary from `/execute`.

**Persona B (working researcher):**
- **Output structure:** IMRAD (Introduction, Methods, Results, Discussion, Conclusion), venue-appropriate.
  - Journal article: full IMRAD + Abstract
  - Conference paper: IMRAD with abstract + references
  - Workshop paper: streamlined IMRAD (shorter Methods, focused Results)

- **Methods section:** pulls directly from pre-registration. Cite the preregistration hash in a footnote: "This analysis was preregistered (OSF ID: ..., SHA256: ...)."
- **Results section:**
  - Confirmatory claims first, in the order preregistered
  - Mark any deviation from prediction with `[honest-pivot: deviation noted]`
  - Exploratory findings in a separate subsection labeled `[EXPLORATORY]`
  - Figures and tables inline; caption includes effect size + CI
- **Discussion section:** interprets findings, addresses limitations (from critique), connects to prior work.
- **Tables and figures:** publication quality, generated by `figure-designer`. APA-formatted captions.

**Persona C (reviewer):**
- N/A — reviewers do not draft. They evaluate via `/critique`.

## Integration with upstream artifacts

**Pre-registration:** pulls the locked pre-registration as the Methods section. The hash is cited as proof of preregistration.

**Results from /execute:** integrates point estimates, CIs, effect sizes, figures. Flags deviations via `honest-pivot` in the draft itself so the reader sees the change.

**Critique resolutions:** all BLOCKs must be fixed before entering /draft. WAARNs are carried forward as revision notes (visible in margins if using DOCX).

**Exploratory labeling:** every result not in the preregistration is explicitly tagged `[EXPLORATORY]` in the draft.

## Failure modes to refuse

- **User wants to reframe exploratory findings as confirmatory.** Mode refuses and points to the preregistration: "This finding wasn't in the locked preregistration, so it's exploratory. You can't change the preregistration retroactively. Would you like to acknowledge it as exploratory and discuss what future confirmation would look like?"
- **Results contradict preregistration predictions severely.** Mode honors the honest-pivot clause: "Your preregistration predicted +3pp on AIME. Your actual result was +2.1pp at p=0.073. This is a genuine deviation. I'm marking it clearly in the draft under [honest-pivot: deviation noted] so readers can see it."
- **User wants to omit inconvenient figures or citations.** Mode includes them anyway (with a note in margins): "This result was in the preregistration. Omitting it weakens the defensibility of your claim. I've included it here; let me know if you want to discuss."

## Output quality gates

Before handoff to `/publish`:

- [ ] All [cite: ...] tags resolved (no orphan citations)
- [ ] All figures have captions with effect sizes and CIs (if quantitative)
- [ ] All [EXPLORATORY] tags correctly applied (matches preregistration)
- [ ] Word count within recipe guidelines (e.g., patent brief 2–5 pages)
- [ ] Methods section cites preregistration hash
- [ ] No overclaiming language (no "substantially" without evidence, no "proves" without p-values)
- [ ] Limitations section acknowledges critique findings
