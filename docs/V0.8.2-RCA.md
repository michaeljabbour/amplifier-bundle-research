# v0.8.2 — Root Cause Analysis + Fix Plan

**Status:** in progress (2026-04-30)
**Triggered by:** post-ship audit of v0.8.0/v0.8.1 by `recipes:recipe-author` and `amplifier:amplifier-expert`. All 5 new recipes failed `recipes operation=validate`. One agent name collision detected. Six silent-failure bugs (approval gates, custom top-level keys, etc.).

This document is the durable RCA. It explains how broken recipes shipped, what the actual bugs are, the fix plan, the test gate that prevents recurrence, and the incident-log framing for the v0.8.0/v0.8.1 tags on GitHub.

---

## RCA — How broken recipes shipped

The chain of mistakes:

1. **Trusted the wrong source as authoritative.** I authored `recipes/orchestrated-loop.yaml` based on an excerpt in the v3 program paper at `~/dev/amplifier-scientist/papers/v3-2026-04-29/`. That excerpt used `type: while`, `body:`, `inputs:`, `outputs:` — none of which are real recipe-engine primitives. The excerpt was **aspirational shape**, not validated YAML. I should have validated against `@recipes:docs/RECIPE_SCHEMA.md` and `@recipes:examples/attractor/attractor.yaml` (the canonical reference) instead.

2. **Did not run the actual validator.** `amplifier tool invoke recipes operation=validate recipe_path=...` exists. I never ran it. A 10-second check would have caught all the schema violations before the first commit.

3. **Healthcheck verified the wrong thing.** `scripts/v0.8.0-healthcheck.sh` and `scripts/v0.8.1-healthcheck.sh` checked file presence (`[ -f ... ]`) and string presence (`grep -q ...`). They did NOT check recipe schema validity. Their 43/43 and 34/34 green-light results gave false confidence: the files existed with the right strings; they were not parseable by the engine.

4. **Conflated agent author transfer with namespace planning.** I copied `paper-architect.md` from `amplifier-bundle-scientificpaper` verbatim, including its `meta: name: paper-architect`. When both bundles are installed simultaneously the resolver sees two agents with the same name and last-mount-wins silently. Should have renamed to `research-paper-architect` on transfer.

5. **Silent-failure bugs reinforced false confidence.** Several of the bugs (`companion_recipes:`, `exit_codes:`, flat `approval_required:`, top-level `cache_only:`) **pass the validator silently** because the `Recipe` and `Stage` dataclasses drop unknown fields rather than rejecting them. So even if I had run the validator, some bugs would not have surfaced — only the kwarg-error bugs would have. The semantic bugs (silently-disabled approval gates, silently-ignored top-level keys) would have shipped anyway.

The forcing function: any new recipe in this bundle from now on must pass `recipes operation=validate` AND be inspected by a checklist that catches the silent-failure bugs (flat approval keys, custom top-level keys, plural `outputs:`, `type: while/llm_judge/shell`, `inputs:` on sub-recipe steps).

## The bugs to fix

### Blocking (recipe will not parse, validator returns TypeError)

| # | Bug | Files | Fix |
|---|---|---|---|
| 1 | `type: while` on stage | `orchestrated-loop.yaml` | Use canonical pattern: one `type: recipe` step with `while_condition + max_while_iterations + update_context` (per `attractor.yaml` Stage 3). Extract iteration body into `recipes/orchestrated-loop-iteration.yaml`. |
| 2 | `type: llm_judge` on step | `orchestrated-loop.yaml` | Replace with `type: agent` + `provider_preferences:` (cross-vendor-judge enforcement). |
| 3 | `type: shell` on steps | `cache-only-verify.yaml` (×6) | `s/type: shell/type: bash/g` |
| 4 | `inputs:` on `type: recipe` steps | `orchestrated-loop.yaml`, `idea-generation.yaml` | Rename to `context:` |
| 5 | `outputs:` (plural list) on steps | `orchestrated-loop.yaml`, `bundle-overlay-proposer.yaml`, `residual-adjudicator.yaml`, `idea-generation.yaml` | Replace with `output:` (singular string); use `parse_json: true` if structured output needed |
| 6 | `body:` and `initial_state:` on stage | `orchestrated-loop.yaml` | Drop both — stages have only `name`, `steps`, `approval`. Initial state goes in top-level `context:`; mutation via `update_context:` on the while step. |
| 7 | `{{var | join(' ')}}` Jinja-style filter | `cache-only-verify.yaml` | Engine has no filters. Rewrite as a pure shell loop. |

### Silent failure (parses clean but doesn't do what the YAML claims)

| # | Bug | Files | Fix |
|---|---|---|---|
| 8 | Flat `approval_required: true` + `approval_prompt: \|` at stage level | `orchestrated-loop.yaml` (3 stages: pre-registration, scaffold, synthesis), `idea-generation.yaml` (1 stage: sharpening) | Replace with nested `approval: { required: true, prompt: \|..., timeout: 0, default: deny }` |
| 9 | Top-level `cache_only:` and `judge_cross_vendor_required:` keys | All 9 baseline recipes (`empirical-paper`, `benchmark-paper`, `replication-study`, `literature-review`, `patent-brief`, `policy-brief`, `white-paper`, `grant-proposal`, `paperbanana-figure`) | Move INTO each recipe's `context:` block so they are at least reachable as substitutable variables |
| 10 | `companion_recipes:`, `exit_codes:`, `publish_gates:` top-level keys | `orchestrated-loop.yaml`, `idea-generation.yaml` | These are silently dropped by the engine; either delete them or convert to `# comment` documentation. We'll keep them as block comments above the recipe header so the documentation value is preserved without the misleading impression that the engine enforces them. |
| 11 | `update-loop-state` agent step that "mutates" parent context | `orchestrated-loop.yaml` | Agents return strings under `output:`; they cannot mutate parent loop state. Replace with `update_context:` block on the while step (canonical attractor pattern). |
| 12 | Stage-level `description:` and `optional:` keys | `cache-only-verify.yaml` | Both silently dropped. Move stage descriptions into `# comment` headers; replace `optional: true` with `condition:` on inner steps or `\|\| true` in bash. |

### Namespace collision

## Fix plan (sequence)

1. **Sanity check the validator** on a known-good recipe (e.g., `recipes/empirical-paper.yaml`) to confirm the `recipes operation=validate` tool is accessible and returns the expected output.
2. **Apply Bug 13** (agent rename) — straightforward sed + bundle.md edit.
3. **Apply Bug 3, 7, 12** to `cache-only-verify.yaml` — local fixes, easiest recipe.
4. **Apply Bug 5** to `bundle-overlay-proposer.yaml` and `residual-adjudicator.yaml` — outputs plural→singular only.
5. **Apply Bug 5 + Bug 8** to `idea-generation.yaml`.
6. **Rewrite `orchestrated-loop.yaml` from scratch** in the canonical attractor pattern: extract the iteration body into a new `recipes/orchestrated-loop-iteration.yaml` sub-recipe; main recipe is one `type: recipe` step with `while_condition`, `max_while_iterations`, `update_context`. This is the largest fix.
7. **Apply Bug 9** — move top-level `cache_only:` and `judge_cross_vendor_required:` into each recipe's `context:` block (12 files: 9 baseline + 3 with judge_cross_vendor).
8. **Validate every new recipe** via `recipes operation=validate`. Every recipe must return valid before continuing.
9. **Update healthcheck** to v0.8.2 — add a `recipes operation=validate` step for every new recipe in the bundle. Rename to `scripts/v0.8.2-healthcheck.sh`.
10. **Document the incident** in `docs/V0.8.0-PLAN.md` — add an "Incident log" section explaining v0.8.0/v0.8.1 superseded by v0.8.2 due to schema violations.
11. **Clean up `.schema-probe/`** the recipe-author left behind.
12. **Push v0.8.2** via `foundation:git-ops` — single commit, tagged `v0.8.2`. Do NOT force-move `v0.8.0` or `v0.8.1` tags. Update GitHub release notes for the broken tags to flag them as superseded.

## Test gate (preventing recurrence)

The new healthcheck section that v0.8.2 introduces:

```bash
echo "Recipe schema validation (gate against v0.8.0/v0.8.1-class bugs):"
for r in recipes/orchestrated-loop.yaml \
         recipes/orchestrated-loop-iteration.yaml \
         recipes/bundle-overlay-proposer.yaml \
         recipes/residual-adjudicator.yaml \
         recipes/idea-generation.yaml \
         recipes/cache-only-verify.yaml; do
    if amplifier tool invoke recipes operation=validate recipe_path="$r" 2>&1 | grep -q '"status": "valid"'; then
        ok "  $r passes recipes validate"
    else
        no "  $r FAILS recipes validate"
    fi
done
```

This catches the kwarg-error bugs (Bugs 1-7). It does NOT catch the silent-failure bugs (Bugs 8-12), because those parse clean. For those, the v0.8.2 healthcheck adds a string-presence check for the canonical patterns:

```bash
echo "Canonical pattern presence (gate against silent-failure bugs):"
contains "approval:" "recipes/orchestrated-loop.yaml"     "orchestrated-loop has nested approval blocks"
contains_e "context:" "recipes/orchestrated-loop.yaml"     "orchestrated-loop has top-level context block"
not_contains "type: while" "recipes/orchestrated-loop.yaml"  "orchestrated-loop does NOT use type: while"
not_contains "type: llm_judge" "recipes/orchestrated-loop.yaml" "orchestrated-loop does NOT use type: llm_judge"
not_contains "approval_required:" "recipes/orchestrated-loop.yaml" "orchestrated-loop does NOT use flat approval_required"
not_contains "outputs:" "recipes/orchestrated-loop.yaml"   "orchestrated-loop does NOT use plural outputs:"
```

Every new recipe added to the bundle going forward must pass both the `recipes validate` gate and the canonical-pattern gate.
